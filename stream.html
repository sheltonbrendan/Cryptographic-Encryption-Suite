How LFSR Works

     
  • Linear Feedback Shift Register uses a binary seed and tap positions
    •  
  • Each clock cycle shifts bits and XORs selected positions to generate feedback
    •  
  • Produces a deterministic but pseudorandom bit sequence
    •  
  • Period length depends on tap selection — maximum length requires primitive polynomials

How RC4 Works

     
  • RC4 uses a key-scheduling algorithm to initialise a 256-byte state array
    •  
  • A pseudorandom generation algorithm produces a keystream byte by byte
    •  
  • Each plaintext byte is XORed with a keystream byte to produce ciphertext
    •  
  • Decryption is identical — XOR the ciphertext with the same keystream
    •  
  • RC4 is now considered cryptographically weak and deprecated in modern systems
